## 一、角色设定技巧

### 1.1 为什么需要角色设定

角色设定（Role Prompting）是提示词工程中最有效的技巧之一。通过为模型分配特定角色，可以：
- **激活领域知识**：模型会调用与该角色相关的专业知识
- **调整语言风格**：专家 vs 科普作家，语气完全不同
- **设定回答视角**：技术视角 vs 用户视角 vs 商业视角

### 1.2 角色设定的基本格式

```
你是一位[专业身份]，拥有[相关经验/背景]。

你的任务是[具体任务]。
```

### 1.3 角色设定实例对比

**❌ 无角色设定**
```
解释一下什么是 REST API。
```

**✅ 有角色设定**
```
你是一位有 10 年经验的后端架构师，擅长设计和解释 API 规范。

请向一位初级开发者解释什么是 REST API，要求：
1. 使用通俗易懂的语言
2. 包含实际代码示例
3. 说明 REST 的核心原则
```

### 1.4 常见角色类型

| 场景 | 推荐角色 | 效果 |
|------|----------|------|
| 代码相关 | 资深软件工程师、架构师、代码审查专家 | 输出更专业、规范 |
| 文案创作 | 资深文案、营销专家、科技作家 | 语言更精炼、有吸引力 |
| 学术解释 | 大学教授、研究员 | 解释更深入、严谨 |
| 教学辅导 | 耐心的导师、培训师 | 解释更循序渐进 |
| 商业分析 | 商业分析师、咨询顾问 | 分析更全面、有洞察力 |
| 翻译任务 | 专业翻译、本地化专家 | 译文更地道 |

### 1.5 角色设定进阶技巧

**多角色协作**
```
请从以下三个角色的视角，分别评价这个产品设计：

1. **产品经理**：关注用户需求和商业价值
2. **技术负责人**：关注技术可行性和实现成本
3. **用户体验设计师**：关注交互流程和易用性

每个角色的评价控制在 200 字以内。
```

**角色属性细化**
```
你是一位专注于 Python 性能优化的专家，具有以下特点：
- 偏爱使用内置函数和标准库
- 注重代码的可读性和可维护性
- 熟悉 CPython 的内部实现
- 经常使用 cProfile 和 timeit 进行性能分析

请审查以下 Python 代码，找出性能瓶颈并提供优化建议。
```

## 二、任务描述的清晰表达

### 2.1 任务描述的核心原则

1. **具体而非模糊**
   - ❌ "写一篇好文章"
   - ✅ "写一篇 800 字的科技评论文章，分析 AI 对编程教育的影响"

2. **可量化而非主观**
   - ❌ "写得详细一些"
   - ✅ "包含至少 3 个具体案例，每个案例分析不少于 100 字"

3. **可操作而非抽象**
   - ❌ "让代码更高效"
   - ✅ "将时间复杂度从 O(n²) 优化到 O(n log n)"

### 2.2 任务描述的完整结构

一个完整的任务描述应包含：

```
【任务类型】：生成/分析/转换/总结/解释...
【处理对象】：具体的数据/文本/代码...
【具体要求】：格式、长度、风格、要点...
【参考标准】：质量要求、验收标准...
```

### 2.3 实例：邮件撰写任务

**❌ 模糊的任务描述**
```
帮我写一封邮件，通知大家明天开会。
```

**✅ 清晰的任务描述**
```
请撰写一封正式的会议通知邮件。

**背景**：
- 收件人：技术团队全体成员（约 20 人）
- 会议主题：Q4 技术规划评审
- 时间：明天（2024 年 1 月 15 日）下午 2:00
- 地点：会议室 A 或腾讯会议（ID: 123456789）
- 时长：预计 2 小时

**要求**：
1. 邮件主题明确，包含会议主题和时间
2. 正文包含会议议程的简要说明
3. 提醒参会者提前准备 Q3 工作总结
4. 语气正式但友好
5. 总字数控制在 200-300 字
```

## 三、分隔符的正确使用

### 3.1 为什么要使用分隔符

分隔符（Delimiters）可以帮助模型：
- 清晰区分不同部分的内容
- 避免指令与待处理内容混淆
- 处理包含特殊字符的输入

### 3.2 常用分隔符

| 分隔符 | 使用场景 |
|--------|----------|
| ``` | 代码块、长文本 |
| """ | 长文本、多行字符串 |
| --- | 章节分隔 |
| ### | 小节分隔 |
| XML 标签 | 结构化数据 |

### 3.3 分隔符使用实例

**代码审查场景**
```
请审查以下 Python 函数，找出其中的 bug。

代码：
```python
def calculate_average(numbers):
    total = 0
    for num in numbers:
        total += num
    return total / len(numbers)
```

要求：
1. 指出具体问题
2. 说明可能的后果
3. 提供修复后的代码
```

**XML 标签分隔**
```
请将以下用户反馈分类到对应的类别中。

<feedback>
这个产品太难用了，根本找不到想要的功能！
</feedback>

<categories>
1. 功能建议
2. 使用问题
3. 性能问题
4. 界面设计
5. 其他
</categories>

请只输出类别编号。
```

### 3.4 分隔符使用技巧

1. **选择不会出现在内容中的分隔符**
   ```
   ❌ 如果处理 Python 代码，不要用 ``` 作为分隔符
   ✅ 可以使用 <code> 标签或自定义标记
   ```

2. **保持分隔符的一致性**
   ```
   ✅ 整个 Prompt 中使用同一种风格的分隔符
   ```

3. **明确标记内容类型**
   ```
   <input type="code" language="python">
   def hello():
       print("Hello")
   </input>
   ```

## 四、输出格式的精确控制

### 4.1 格式控制的重要性

- **便于解析**：结构化输出易于程序处理
- **提高可读性**：清晰的格式提升用户体验
- **保持一致性**：确保多次输出的格式统一

### 4.2 常用输出格式

#### Markdown 格式
```
请按以下格式输出分析结果：

## 问题概述
简要描述核心问题

## 详细分析
### 方面一
详细说明...

### 方面二
详细说明...

## 建议方案
1. 建议一
2. 建议二
```

#### JSON 格式
```
请将分析结果输出为 JSON 格式：

{
  "summary": "问题概述",
  "severity": "高/中/低",
  "issues": [
    {
      "title": "问题标题",
      "description": "问题描述",
      "suggestion": "改进建议"
    }
  ],
  "recommendations": ["建议1", "建议2"]
}
```

#### 表格格式
```
请用表格形式对比以下三个方案：

| 维度 | 方案 A | 方案 B | 方案 C |
|------|--------|--------|--------|
| 成本 | ... | ... | ... |
| 实施难度 | ... | ... | ... |
| 预期效果 | ... | ... | ... |
```

### 4.3 格式控制实例

**情感分析任务**
```
请分析以下评论的情感倾向。

评论：这家餐厅的服务真的太棒了，食物也很美味！

输出格式要求：
1. 情感标签：[积极 / 消极 / 中性]
2. 置信度：0-1 之间的数值
3. 关键词：列出 3 个最能体现情感的关键词
4. 理由：简要说明判断依据（50 字以内）

示例输出：
```
情感标签：积极
置信度：0.95
关键词：服务棒、食物美味、太棒了
理由：评论使用了多个正面词汇表达对服务和食物的满意
```
```

### 4.4 结构化输出技巧

**使用占位符**
```
请填写以下模板：

【核心观点】：[一句话总结核心观点]

【支撑论据】：
1. [论据一]
2. [论据二]
3. [论据三]

【潜在反驳】：[列出可能的反对意见]

【结论】：[总结性陈述]
```

**指定数据类型**
```
请按以下要求输出：
- score: 整数，0-100
- tags: 字符串数组，最多 5 个
- summary: 字符串，100-150 字
- valid: 布尔值，表示结果是否有效
```

## 五、约束条件的合理设置

### 5.1 约束条件的类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 长度约束 | 控制输出长度 | "不超过 200 字"、"至少 3 点" |
| 格式约束 | 指定输出格式 | "用表格输出"、"JSON 格式" |
| 内容约束 | 限定内容范围 | "只讨论技术层面"、"不包含代码" |
| 风格约束 | 指定语言风格 | "用通俗易懂的语言"、"学术风格" |
| 排除约束 | 明确不要什么 | "不要出现专业术语"、"不提出建议" |

### 5.2 约束条件设置实例

**代码生成任务**
```
请编写一个 Python 函数，实现以下功能：
- 功能：计算斐波那契数列的第 n 项
- 约束条件：
  1. 时间复杂度不超过 O(n)
  2. 不使用递归
  3. 处理 n <= 0 的边界情况
  4. 添加函数文档字符串（docstring）
  5. 包含 3 个测试用例
```

**文案创作任务**
```
请为一款新的健身 App 撰写宣传文案。

约束条件：
- 目标受众：25-35 岁的上班族
- 文案长度：50-80 字
- 风格：积极向上，避免使用"减肥""瘦身"等词汇
- 必须包含：App 名称"FitLife"
- 卖点：每天只需 15 分钟
```

### 5.3 约束条件的平衡

**过多约束的问题**
```
❌ 约束过多可能导致输出僵硬
请写一篇关于人工智能的文章，要求：
1. 正好 500 字
2. 必须提到"深度学习""神经网络""Transformer"这三个词
3. 要用"首先""其次""最后"的结构
4. 第二段必须引用专家观点
5. 不能出现"AI"这个缩写
6. 要用拟人化手法
7. 要包含一个类比
8. 结尾要有悬念
```

**合理设置约束**
```
✅ 核心约束优先
请写一篇关于人工智能的科普文章，要求：
1. 目标读者：高中生
2. 篇幅：400-600 字
3. 必须解释清楚：AI 能做什么、不能做什么
4. 语言风格：通俗易懂，避免过多专业术语
```

## 六、上下文提供策略

### 6.1 为什么需要上下文

- 帮助模型理解背景
- 减少歧义和误解
- 获得更精准的回答

### 6.2 上下文的提供方式

**背景信息**
```
**项目背景**：
我们是一个 5 人的初创团队，正在开发一个面向大学生的在线学习平台。
目前技术栈：React + Node.js + MongoDB
团队经验：大部分成员有 1-2 年工作经验

**当前问题**：
我们正在考虑是否引入 TypeScript，需要权衡利弊...

请从技术选型角度给出建议。
```

**前置对话**
```
之前的讨论：
用户：我想学习编程，有什么建议？
助手：你之前有编程基础吗？
用户：完全没有，是零基础。

基于以上对话，请给出适合零基础学习者的编程入门建议。
```

### 6.3 上下文的精简原则

- **相关性**：只提供与任务相关的信息
- **时效性**：提供最新的信息
- **简洁性**：避免冗长，关键信息优先

## 七、实战：Prompt 优化工作坊

### 案例 1：代码解释

**原始 Prompt**
```
解释一下这段代码。
```

**优化过程**

v1.0 - 添加角色和语言
```
你是一位经验丰富的 Python 开发者。请解释以下代码的功能。
```

v2.0 - 添加具体任务
```
你是一位经验丰富的 Python 开发者。

请解释以下代码：
1. 这段代码实现了什么功能？
2. 关键逻辑是什么？
3. 使用了哪些 Python 特性？
```

v3.0 - 添加输出格式
```
你是一位经验丰富的 Python 开发者，擅长向初学者解释代码。

请分析以下代码，并按以下格式输出：

**功能概述**：一句话描述代码功能

**逐行解析**：
- 第 X 行：[代码] - 作用说明

**涉及的知识点**：
- 知识点 1：简要说明
- 知识点 2：简要说明

**类似应用**：举 2 个类似的实际应用场景
```

### 案例 2：数据分析

**原始 Prompt**
```
分析这组数据。
```

**优化后的 Prompt**
```
你是一位数据分析师，擅长从数据中发现洞察。

**数据说明**：
以下是一家电商网站过去 7 天的销售数据：
```
日期,订单数,销售额,访客数
2024-01-01,150,￥45000,3000
2024-01-02,180,￥54000,3600
...
```

**分析任务**：
1. 计算每日的转化率（订单数/访客数）
2. 识别数据中的趋势和异常
3. 找出客单价的变化规律

**输出要求**：
- 用表格展示计算结果
- 列出 3 个关键发现
- 提出 2 条可执行的业务建议
- 总字数控制在 500 字以内
```

## 总结

本文详细介绍了提示词工程的基础写作技巧：

1. **角色设定**：通过分配专业角色激活领域知识，调整输出风格
2. **任务描述**：具体、可量化、可操作的描述方式
3. **分隔符使用**：```、"""、XML 标签等分隔符的正确使用
4. **输出格式控制**：Markdown、JSON、表格等格式的精确控制
5. **约束条件设置**：长度、格式、内容、风格等约束的合理平衡
6. **上下文提供**：背景信息的有效传递

掌握这些基础技巧后，你已经能够写出清晰、有效的 Prompt。在下一篇文章中，我们将探讨更进阶的技巧——**上下文学习策略**，包括 Few-shot、Zero-shot 等模式。
