## 前言

容错机制是 Flink 实现高可用的核心。通过 Checkpoint 和 Savepoint，Flink 能够在故障发生时恢复状态并继续处理，同时保证精确一次语义。

### Barrier 对齐机制

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Barrier 对齐机制                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  Checkpoint Barrier 是 Checkpoint 的核心机制：                           │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  Barrier 流动过程：                                             │   │
│  │                                                                 │   │
│  │  Source                    Map                   Sink           │   │
│  │  ┌─────┐               ┌─────┐                ┌─────┐          │   │
│  │  │Barrier              │     │                │     │          │   │
│  │  │ ──► │ ───────────► │Bar│ │ ───────────► │Bar│ │          │   │
│  │  │     │               │    ││               │    ││          │   │
│  │  │Data │               │Data││               │Data││          │   │
│  │  └─────┘               └─────┘                └─────┘          │   │
│  │                                                                 │   │
│  │  步骤：                                                         │   │
│  │  1. JobManager 触发 Checkpoint，注入 Barrier                   │   │
│  │  2. Barrier 随数据流流动                                        │   │
│  │  3. 算子收到 Barrier 后，触发状态快照                           │   │
│  │  4. 所有算子完成快照后，Checkpoint 完成                        │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  Barrier 对齐（Exactly-Once）：                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  多输入场景下的 Barrier 对齐：                                  │   │
│  │                                                                 │   │
│  │         Input 1: ──Data──Barrier──Data──►                      │   │
│  │                        │                                       │   │
│  │                        ▼ 等待                                  │   │
│  │         Input 2: ──Data──Data──Barrier──Data──►                │   │
│  │                              │                                 │   │
│  │                              ▼ 对齐                            │   │
│  │                              [触发快照]                        │   │
│  │                                                                 │   │
│  │  规则：必须等待所有输入的 Barrier 到达后才触发快照              │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### Checkpoint 配置

```java
// 启用 Checkpoint（每 1 分钟）
env.enableCheckpointing(60000);

// Checkpoint 配置
CheckpointConfig config = env.getCheckpointConfig();

// 设置模式为精确一次
config.setCheckpointingMode(CheckpointingMode.EXACTLY_ONCE);

// Checkpoint 超时时间
config.setCheckpointTimeout(60000);

// 最大并发 Checkpoint 数
config.setMaxConcurrentCheckpoints(1);

// 最小间隔（两次 Checkpoint 之间）
config.setMinPauseBetweenCheckpoints(500);

// 故障时是否保留 Checkpoint
config.setExternalizedCheckpointCleanup(
    ExternalizedCheckpointCleanup.RETAIN_ON_CANCELLATION);

// 容忍的 Checkpoint 失败次数
config.setTolerableCheckpointFailureNumber(3);
```

## Savepoint

### Savepoint 与 Checkpoint 对比

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Savepoint vs Checkpoint                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  特性              Checkpoint           Savepoint               │   │
│  │  ────────────────────────────────────────────────────────────── │   │
│  │  触发方式          自动（定时）          手动                    │   │
│  │  生命周期          可被自动删除          永久保存                │   │
│  │  用途              故障恢复              版本升级、迁移          │   │
│  │  格式              可能依赖实现          标准格式                │   │
│  │  兼容性            低                    高                      │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  Savepoint 使用场景：                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • 应用版本升级                                                 │   │
│  │  • 集群迁移                                                     │   │
│  │  • Flink 版本升级                                               │   │
│  │  • A/B 测试                                                     │   │
│  │  • 扩缩容                                                       │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  Savepoint 操作：                                                       │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  # 触发 Savepoint                                               │   │

│  │  flink run -s <savepoint-path> -d <jar-file>                    │   │
│  │                                                                 │   │
│  │  # 取消作业并创建 Savepoint                                     │   │
│  │  flink cancel -s [target-directory] <job-id>                    │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
```

## 两阶段提交协议

### 2PC 实现 Exactly-Once

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        两阶段提交协议                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  端到端 Exactly-Once 需要两阶段提交协议：                                 │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  TwoPhaseCommitSinkFunction 接口：                              │   │
│  │                                                                 │   │
│  │  ┌───────────────────────────────────────────────────────────┐ │   │
│  │  │                                                           │ │   │
│  │  │  Phase 1: 预提交（Pre-Commit）                             │ │   │
│  │  │  ─────────────────────────────────────                    │ │   │
│  │  │  1. beginTransaction(): 开始事务                          │ │   │
│  │  │  2. invoke(): 写入数据（但未提交）                        │ │   │
│  │  │  3. preCommit(): 预提交                                   │ │   │
│  │  │                                                           │ │   │
│  │  │  Checkpoint 完成                                          │ │   │
│  │  │  ────────────────                                         │ │   │
│  │  │                                                           │ │   │
│  │  │  Phase 2: 正式提交（Commit）                               │ │   │
│  │  │  ─────────────────────────────                            │ │   │
│  │  │  4. commit(): 正式提交事务                                │ │   │
│  │  │  5. abort(): 回滚（如果 Checkpoint 失败）                  │ │   │
│  │  │                                                           │ │   │
│  │  └───────────────────────────────────────────────────────────┘ │   │
│  │                                                                 │   │
│  │  时间线：                                                       │   │
│  │  ┌───────────────────────────────────────────────────────────┐ │   │
│  │  │                                                           │ │   │
│  │  │  数据写入 ──► Checkpoint 开始 ──► 预提交 ──► CP 完成       │ │   │
│  │  │     │                                       │              │ │   │
│  │  │     │                                       ▼              │ │   │
│  │  │     │                              正式提交 ──► 数据可见   │ │   │
│  │  │                                                           │ │   │
│  │  └───────────────────────────────────────────────────────────┘ │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  Kafka Sink 示例：                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  KafkaSink<String> sink = KafkaSink.<String>builder()          │   │
│  │      .setBootstrapServers("kafka:9092")                         │   │
│  │      .setRecordSerializer(...)                                  │   │
│  │      .setDeliveryGuarantee(DeliveryGuarantee.EXACTLY_ONCE)     │   │
│  │      .setTransactionalIdPrefix("flink-kafka")                  │   │
│  │      .build();                                                  │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 故障恢复

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        故障恢复流程                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  TaskManager 故障恢复：                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  1. JobManager 检测到 TaskManager 故障                         │   │
│  │  2. 取消所有受影响的 Task                                       │   │
│  │  3. 从最近的 Checkpoint 恢复状态                                │   │
│  │  4. 重新调度 Task 到其他 TaskManager                           │   │
│  │  5. 从 Checkpoint 位置重新消费数据                              │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  JobManager 故障恢复（HA）：                                             │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  1. 多个 JobManager（Leader + Standby）                        │   │
│  │  2. Leader 选举机制（ZooKeeper / Kubernetes）                   │   │
│  │  3. Leader 故障后，Standby 接管                                 │   │
│  │  4. 从共享存储恢复 JobGraph 和 Checkpoint                       │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 总结

本章深入解析了 Flink 容错机制：

| 概念 | 说明 |
|------|------|
| Checkpoint | 自动触发的状态快照 |
| Barrier | Checkpoint 标记，随数据流流动 |
| Savepoint | 手动触发的状态快照，用于升级迁移 |
| 两阶段提交 | 实现端到端 Exactly-Once |
