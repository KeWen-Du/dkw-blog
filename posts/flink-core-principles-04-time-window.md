## 时间语义

### 三种时间语义

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Flink 时间语义                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  1. Processing Time（处理时间）                                 │   │
│  │  ┌───────────────────────────────────────────────────────────┐ │   │
│  │  │                                                           │ │   │
│  │  │  • 数据被处理时的机器时间                                  │ │   │
│  │  │  • 最简单，延迟最低                                        │ │   │
│  │  │  • 不确定性：依赖处理速度、网络延迟                        │ │   │
│  │  │  • 适用于：实时监控、低延迟要求场景                        │ │   │
│  │  │                                                           │ │   │
│  │  └───────────────────────────────────────────────────────────┘ │   │
│  │                                                                 │   │
│  │  2. Event Time（事件时间）                                      │   │
│  │  ┌───────────────────────────────────────────────────────────┐ │   │
│  │  │                                                           │ │   │
│  │  │  • 数据本身携带的时间戳                                    │ │   │
│  │  │  • 确定性：结果不依赖处理顺序                              │ │   │
│  │  │  • 需要 Watermark 配合                                     │ │   │
│  │  │  • 适用于：乱序数据处理、精确结果要求                      │ │   │
│  │  │                                                           │ │   │
│  │  └───────────────────────────────────────────────────────────┘ │   │
│  │                                                                 │   │
│  │  3. Ingestion Time（摄入时间）                                  │   │
│  │  ┌───────────────────────────────────────────────────────────┐ │   │
│  │  │                                                           │ │   │
│  │  │  • 数据进入 Flink 时的机器时间                             │ │   │
│  │  │  • 介于 Processing Time 和 Event Time 之间                 │ │   │
│  │  │                                                           │ │   │
│  │  └───────────────────────────────────────────────────────────┘ │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  时间语义选择：                                                         │
│  • 需要精确结果：Event Time                                            │
│  • 需要低延迟：Processing Time                                         │
│  • 乱序数据：Event Time                                                │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### Event Time 示例

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Event Time 处理示例                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  场景：传感器数据，按 5 分钟窗口聚合                                     │
│                                                                         │
│  数据流（实际到达顺序）：                                               │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  到达时间: 10:00:01  10:00:02  10:00:03  10:00:04  10:00:05   │   │
│  │  事件时间: 10:05:30  10:04:20  10:06:10  10:03:50  10:05:45   │   │
│  │  传感器ID:    S1        S2        S1        S2        S3       │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  Processing Time 窗口（错误）：                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  按 10:00:00 - 10:05:00 窗口分组                               │   │
│  │  问题：10:03:50 的事件被错误地分到前一窗口                     │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  Event Time 窗口（正确）：                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  按 事件时间 窗口分组                                           │   │
│  │  10:05:30, 10:05:45 → 窗口 [10:05:00 - 10:10:00)               │   │
│  │  10:04:20 → 窗口 [10:00:00 - 10:05:00)                         │   │
│  │  10:06:10 → 窗口 [10:05:00 - 10:10:00)                         │   │
│  │  10:03:50 → 窗口 [10:00:00 - 10:05:00)                         │   │
│  │                                                                 │   │
│  │  Watermark 到达后，窗口才触发计算                              │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## Watermark 机制

### Watermark 概念

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Watermark 概念                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  Watermark 是 Flink 测量 Event Time 进度的机制：                         │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  Watermark(t) 声明：                                            │   │
│  │  "时间 t 之前的所有数据都已经到达"                              │   │
│  │                                                                 │   │
│  │  数学表示：                                                     │   │
│  │  对于所有事件时间 ≤ t 的数据，都已经被处理                      │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  Watermark 生成策略：                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  1. 周期性 Watermark（Periodic）                                │   │
│  │     • 定期生成（如每 200ms）                                    │   │
│  │     • Watermark = maxTimestamp - outOfOrderness                │   │
│  │                                                                 │   │
│  │  2. 标点式 Watermark（Punctuated）                              │   │
│  │     • 遇到特定事件时生成                                        │   │
│  │     • 适用于某些数据带有特殊标记的场景                          │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  代码示例：                                                             │
│  WatermarkStrategy.forBoundedOutOfOrderness(Duration.ofSeconds(5))     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### Watermark 与窗口触发

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Watermark 与窗口触发                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  窗口：[10:00 - 10:10)，允许延迟 5 秒                                   │
│                                                                         │
│  时间线：                                                               │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  事件:    E1    E2    E3    E4    E5    E6                     │   │
│  │  事件时间: 10:01  10:03  10:07  10:09  10:05  10:12            │   │
│  │                                                                 │   │
│  │  到达顺序: ──►───►───►───►───►───►                             │   │
│  │                                                                 │   │
│  │  Watermark:                                                      │   │
│  │  E1后: W(09:56)  - 不触发                                       │   │
│  │  E2后: W(09:58)  - 不触发                                       │   │
│  │  E3后: W(10:02)  - 不触发                                       │   │
│  │  E4后: W(10:04)  - 不触发                                       │   │
│  │  E5后: W(10:00)  - 不触发 (E5 乱序，不更新 Watermark)           │   │
│  │  E6后: W(10:07)  - 触发！[10:00-10:10) 窗口关闭                 │   │
│  │                                                                 │   │
│  │  规则：Watermark >= 窗口结束时间 时，窗口触发计算               │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  多并行度 Watermark：                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  Watermark 取所有输入的最小值（对齐）                          │   │
│  │                                                                 │   │
│  │  Input 1: W(10:05)                                             │   │
│  │  Input 2: W(10:03)  ──► 系统水位线: W(10:03)                   │   │
│  │  Input 3: W(10:07)                                             │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 窗口类型

```

│                        窗口类型                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. Tumbling Window（滚动窗口）                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  特点：窗口大小固定，不重叠                                     │   │
│  │                                                                 │   │
│  │  ┌─────┐┌─────┐┌─────┐┌─────┐                                 │   │

│  │  └─────┘└─────┘└─────┘└─────┘                                 │   │
│  │  10:00  10:05  10:10  10:15  10:20                            │   │
│  │                                                                 │   │
│  │  代码：.window(TumblingEventTimeWindows.of(Time.minutes(5)))   │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │

│  ┌─────────────────────────────────────────────────────────────────┐   │

│  │    │   W1    │                                                 │   │

│  ┌─────────────────────────────────────────────────────────────────┐   │
```

### 窗口函数

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        窗口函数                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. ReduceFunction（归约函数）                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • 输入输出类型相同                                             │   │
│  │  • 增量计算，效率高                                             │   │
│  │                                                                 │   │
│  │  .reduce((v1, v2) -> v1 + v2)                                  │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  2. AggregateFunction（聚合函数）                                       │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • 输入、累加器、输出类型可不同                                 │   │
│  │  • 增量计算，效率高                                             │   │
│  │                                                                 │   │
│  │  .aggregate(new AggregateFunction<IN, ACC, OUT>() {...})       │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  3. ProcessWindowFunction（窗口处理函数）                               │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • 可访问窗口元数据                                             │   │
│  │  • 可遍历窗口内所有数据                                         │   │
│  │  • 非增量计算，性能较低                                         │   │
│  │                                                                 │   │
│  │  .process(new ProcessWindowFunction<IN, OUT, KEY, W>() {...})  │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  4. 组合使用（推荐）                                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  // 增量聚合 + 窗口函数                                         │   │
│  │  .aggregate(                                                    │   │
│  │      new MyAggregateFunction(),                                 │   │
│  │      new MyProcessWindowFunction()                              │   │
│  │  )                                                              │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 窗口触发器

### Trigger 机制

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Trigger 机制                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  Trigger 决定窗口何时触发计算：                                          │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  TriggerResult:                                                 │   │
│  │  ┌───────────────────────────────────────────────────────────┐ │   │
│  │  │                                                           │ │   │
│  │  │  CONTINUE:  继续等待，不触发                               │ │   │
│  │  │  FIRE:      触发计算，保留窗口内容                         │ │   │
│  │  │  PURGE:     清空窗口内容                                   │ │   │
│  │  │  FIRE_AND_PURGE: 触发计算并清空                            │ │   │
│  │  │                                                           │ │   │
│  │  └───────────────────────────────────────────────────────────┘ │   │
│  │                                                                 │   │
│  │  内置 Trigger:                                                  │   │
│  │  • EventTimeTrigger: Watermark 到达时触发                      │   │
│  │  • ProcessingTimeTrigger: 处理时间到达时触发                   │   │
│  │  • CountTrigger: 元素数量达到阈值时触发                        │   │
│  │  • ContinuousEventTimeTrigger: 连续事件时间触发                │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  自定义 Trigger 示例：                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  .trigger(new Trigger<MyEvent, TimeWindow>() {                  │   │
│  │      @Override                                                  │   │
│  │      public TriggerResult onElement(...) {                      │   │
│  │          // 元素到达时的逻辑                                    │   │
│  │      }                                                          │   │
│  │      @Override                                                  │   │
│  │      public TriggerResult onEventTime(...) {                    │   │
│  │          // 事件时间触发逻辑                                    │   │
│  │      }                                                          │   │
│  │  })                                                             │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 总结

本章深入解析了 Flink 时间语义与窗口机制：

| 概念 | 说明 |
|------|------|
| Event Time | 数据携带的事件时间戳，确定性处理 |
| Processing Time | 处理时的机器时间，低延迟 |
| Watermark | 测量 Event Time 进度，处理乱序 |
| 窗口类型 | 滚动/滑动/会话/全局 |
