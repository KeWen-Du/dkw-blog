## 一、多模态基础概念

### 1.1 什么是多模态

**多模态（Multimodal）** 指的是系统能够同时处理和理解多种类型的数据模态，常见的模态包括：

| 模态 | 数据类型 | 典型任务 |
|------|----------|----------|
| 文本 | 文字、文档 | 理解、生成、翻译 |
| 图像 | 照片、图表、截图 | 识别、描述、分析 |
| 音频 | 语音、音乐 | 转录、识别、合成 |
| 视频 | 动态图像序列 | 理解、摘要、检索 |
| 结构化数据 | 表格、JSON | 分析、查询、可视化 |

## 二、图像理解提示基础

### 2.1 基础图像提示结构

```
[图像]

请描述这张图片中的内容。
```

### 2.2 图像提示的核心要素

一个有效的图像提示应包含：

1. **图像本身**：直接上传或提供 URL
2. **任务指令**：明确要求模型做什么
3. **输出格式**：指定回答的结构
4. **约束条件**：限制或特殊要求

### 2.3 图像描述任务

**基础描述**
```
[上传图片]

请描述这张图片。
```

**结构化描述**
```
[上传图片]

请详细描述这张图片，包括：
1. 主要对象和人物
2. 场景和环境
3. 颜色和光线
4. 整体氛围和情感
5. 任何值得注意的细节
```

**特定元素提取**
```
[上传图片 - 商场内部照片]

请分析这张商场照片，列出：
1. 所有可见的品牌名称
2. 人流量评估（多/中/少）
3. 主要的商品类别
4. 装修风格特点
```

### 2.4 视觉问答（Visual Question Answering, VQA）

```
[上传图片 - 餐厅菜单]

问题：
1. 这家餐厅的主打菜是什么？
2. 最贵的菜品多少钱？
3. 有没有素食选项？
4. 营业时间是什么？

请基于图片内容回答以上问题。
```

## 四、多模态 Chain-of-Thought

### 4.1 为什么需要多模态 CoT

对于复杂的视觉推理任务，模型需要：
1. 先理解图像内容
2. 进行分步推理
3. 结合外部知识
4. 得出最终结论

### 4.2 多模态 CoT 示例

**几何问题求解**
```
[上传图片 - 几何图形题]

题目：如上图所示，在直角三角形 ABC 中，∠C = 90°，
      AC = 3，BC = 4，D 是 AB 的中点。求 CD 的长度。

让我们一步步思考：

第一步：识别图形
从图中可以看出...

第二步：确定已知条件
- 三角形 ABC 是直角三角形
- ∠C = 90°
- AC = 3
- BC = 4
- D 是斜边 AB 的中点

第三步：应用定理
根据直角三角形的性质，斜边中线等于斜边的一半。

第四步：计算斜边长度
AB = √(AC² + BC²) = √(9 + 16) = √25 = 5

第五步：计算 CD 长度
CD = AB / 2 = 5 / 2 = 2.5

答案：CD 的长度是 2.5
```

**图表推理**
```
[上传图片 - 多图表仪表盘]

问题：分析该公司的运营状况，指出主要问题和改进方向。

推理过程：

1. 收入分析
   从收入图表可以看出...
   → 收入呈现上升趋势，但增速放缓

2. 成本分析
   从成本结构图可以看出...
   → 营销成本占比过高，达到 40%

3. 用户分析
   从用户增长图可以看出...
   → 新用户增长放缓，留存率下降

4. 综合判断
   主要问题：获客成本上升，用户留存下降
   根本原因：可能是产品竞争力下降或市场饱和

5. 改进建议
   - 优化产品体验，提高留存率
   - 探索新的获客渠道，降低营销成本
   - 开发新功能或新市场
```

### 6.2 图像检索增强生成

```python
class MultimodalRAG:
    """多模态 RAG 系统"""
    
    def __init__(self, vision_model, text_model, vector_store):
        self.vision_model = vision_model
        self.text_model = text_model
        self.vector_store = vector_store
    
    def query_with_image(self, query_image, text_query=None):
        """
        基于图像的 RAG 查询
        
        Args:
            query_image: 查询图像
            text_query: 可选的文本查询
        """
        # 1. 生成图像描述（用于检索）
        image_description = self.vision_model.describe(query_image)
        
        # 2. 构建检索查询
        if text_query:
            search_query = f"{text_query} {image_description}"
        else:
            search_query = image_description
        
        # 3. 检索相关文档
        retrieved_docs = self.vector_store.search(search_query)
        
        # 4. 构建多模态 Prompt
        prompt = self._build_multimodal_prompt(
            query_image,
            text_query,
            retrieved_docs
        )
        
        # 5. 生成回答
        return self.vision_model.generate(prompt)
    
    def _build_multimodal_prompt(self, image, text_query, documents):
        """构建多模态 RAG Prompt"""
        
        prompt = {
            'images': [image],
            'text': f"""基于提供的参考文档和图像，回答问题。

## 参考文档
{self._format_docs(documents)}

## 用户问题
{text_query or "请分析这张图片"}

请结合图像内容和参考文档回答："""
        }
        
        return prompt
```

## 七、实战案例

### 7.1 智能客服（支持图片上传）

```python
class IntelligentCustomerService:
    def __init__(self, multimodal_llm):
        self.llm = multimodal_llm
    
    def handle(self, user_message, attached_images=None):
        """处理用户消息，支持图文混合输入"""
        
        if attached_images:
            # 多模态处理
            prompt = self._build_multimodal_prompt(
                user_message, 
                attached_images
            )
        else:
            # 纯文本处理
            prompt = user_message
        
        return self.llm.generate(prompt)
    
    def _build_multimodal_prompt(self, message, images):
        """构建多模态客服 Prompt"""
        
        return {
            'images': images,
            'text': f"""你是一位智能客服助手。用户发送了以下消息和图片，
请帮助解决问题。

用户消息：{message}

请分析：
1. 用户遇到了什么问题？
2. 图片展示了什么情况？
3. 提供具体的解决方案
4. 如果需要进一步帮助，询问什么信息？"""
        }
```

## 八、多模态提示最佳实践

### 8.1 通用原则

1. **明确任务类型**
   - 描述、分类、问答、生成？
   - 单图还是多图？

2. **指定输出格式**
   - 结构化输出更易用
   - 明确每项内容的来源

3. **设置约束条件**
   - 基于图像还是结合知识？
   - 不确定时如何处理？

4. **提供上下文**
   - 图像背景信息
   - 用户的真实意图

### 8.2 常见陷阱

**陷阱 1：假设模型能看到细节**
```
❌ 不好的提示
"第三行第二个字是什么？"
（模型可能无法精确定位）

✅ 改进的提示
"图片中有一段文字，请完整转录这段文字，
然后告诉我第三行第二个字是什么。"
```

**陷阱 2：忽略图像质量**
```
❌ 不好的提示
直接上传模糊图片并要求详细分析

✅ 改进的做法
- 提醒用户上传清晰的图片
- 对低质量图片预处理
- 在提示中说明"如果看不清，请说明"
```

**陷阱 3：过度依赖视觉**
```
❌ 不好的做法
只上传图片，不提供任何文字说明

✅ 改进的做法
- 提供图片背景信息
- 明确你想知道什么
- 说明图片的来源和可信度
```

3. **多模态 CoT**：将链式思考扩展到视觉领域，解决复杂的多模态推理问题

4. **视频和音频处理**：了解如何处理时序多模态数据

5. **多模态 RAG**：构建能够检索和利用多模态知识的系统

**核心要点**：
- 多模态提示需要考虑不同模态的信息如何协同
- 清晰的任务定义和输出格式对多模态任务尤为重要
- 多模态能力为 AI 应用开辟了全新的可能性
