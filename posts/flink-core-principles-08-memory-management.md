## 前言

Flink 自主管理内存，避免了 JVM 内存管理的诸多问题。理解 Flink 的内存模型对于调优和排查内存问题至关重要。

### 内存组件详解

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        内存组件详解                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. Framework Heap（框架堆内存）                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • Flink 框架内部使用的堆内存                                   │   │
│  │  • 默认 128MB                                                   │   │
│  │  • 配置: taskmanager.memory.framework.heap.size                 │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  2. Task Heap（任务堆内存）                                             │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • 用户代码和算子使用的堆内存                                   │   │
│  │  • 默认 = Total Flink Memory - 其他组件                        │   │
│  │  • 配置: taskmanager.memory.task.heap.size                      │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  3. Managed Memory（托管内存）                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • Flink 直接管理的堆外内存                                     │   │
│  │  • 用于：RocksDB State Backend、排序、缓存                      │   │
│  │  • 默认占 Total Flink Memory 的 40%                            │   │
│  │  • 配置: taskmanager.memory.managed.fraction                    │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  4. Network Memory（网络内存）                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • 网络缓冲区（Network Buffer）                                 │   │
│  │  • 用于 Task 之间的数据传输                                     │   │
│  │  • 默认占 Total Flink Memory 的 10%                            │   │
│  │  • 配置: taskmanager.memory.network.fraction                    │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 内存配置

### 常用配置

```yaml
# TaskManager 总进程内存
taskmanager.memory.process.size: 4096m

# 或分别配置
taskmanager.memory.flink.size: 3072m
taskmanager.memory.jvm-overhead.size: 512m

# 托管内存占比
taskmanager.memory.managed.fraction: 0.4

# 网络内存占比
taskmanager.memory.network.fraction: 0.1

# 框架堆内存
taskmanager.memory.framework.heap.size: 128m

# 框架堆外内存
taskmanager.memory.framework.off-heap.size: 128m

# JVM Metaspace
taskmanager.memory.jvm-metaspace.size: 256m

# JVM 开销占比
taskmanager.memory.jvm-overhead.fraction: 0.1
```

### 内存计算公式

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        内存计算公式                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  配置优先级：                                                           │
│                                                                         │
│  1. 如果指定了 process.size：                                           │
│     Total Flink Memory = process.size - JVM Overhead - Metaspace       │
│                                                                         │
│  2. 如果指定了 flink.size：                                             │

│  │  Task Heap:          1792 MB (剩余)                             │   │
```

## 托管内存

### 托管内存用途

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        托管内存用途                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  1. RocksDB State Backend                                       │   │
│  │  ┌───────────────────────────────────────────────────────────┐ │   │
│  │  │                                                           │ │   │
│  │  │  • RocksDB 使用托管内存存储状态                            │ │   │
│  │  │  • 每个 Slot 分配 managed_memory / num_slots               │ │   │
│  │  │  • 避免与 JVM 堆竞争                                       │ │   │
│  │  │                                                           │ │   │
│  │  └───────────────────────────────────────────────────────────┘ │   │
│  │                                                                 │   │
│  │  2. 排序和 Hash Join                                            │   │
│  │  ┌───────────────────────────────────────────────────────────┐ │   │
│  │  │                                                           │ │   │
│  │  │  • 批处理中的排序操作使用托管内存                          │ │   │
│  │  │  • Hash Join 的 Hash 表使用托管内存                        │ │   │
│  │  │                                                           │ │   │
│  │  └───────────────────────────────────────────────────────────┘ │   │
│  │                                                                 │   │
│  │  3. Python Process                                              │   │
│  │  ┌───────────────────────────────────────────────────────────┐ │   │
│  │  │                                                           │ │   │
│  │  │  • PyFlink 进程使用托管内存                                │ │   │
│  │  │                                                           │ │   │
│  │  └───────────────────────────────────────────────────────────┘ │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  托管内存优势：                                                         │
│  • 避免长时间 GC                                                        │
│  • 精确控制内存使用                                                     │
│  • 支持大状态处理                                                       │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 内存优化策略

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        内存优化策略                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 状态大 → 增加 Managed Memory                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  taskmanager.memory.managed.fraction: 0.5                      │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  2. 网络传输多 → 增加 Network Memory                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  taskmanager.memory.network.fraction: 0.15                     │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  3. GC 问题 → 使用 RocksDB + Managed Memory                             │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  state.backend: rocksdb                                        │   │
│  │  taskmanager.memory.managed.fraction: 0.4                      │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  4. 大任务堆内存 → 减少 Framework Heap                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  taskmanager.memory.framework.heap.size: 64m                   │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  5. 启用 Buffer Debloat（自适应网络缓冲区）                             │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  taskmanager.network.memory.buffer-debloat.enabled: true       │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 总结

本章深入解析了 Flink 内存管理：

| 概念 | 说明 |
|------|------|
| JVM Heap | 框架堆 + 任务堆 |
| Managed Memory | 托管内存，用于 RocksDB、排序 |
| Network Memory | 网络缓冲区 |
| JVM Overhead | JVM 开销（线程栈、GC 等） |
