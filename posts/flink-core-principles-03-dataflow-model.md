### DataStream 类型

```

│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │

│  │  DataStream<T>                                                  │   │

│  │  ConnectedStream<T1, T2>                                        │   │

│  │  SideOutputDataStream<T>                                        │   │

│  │  │ • 侧输出流                                                 │ │   │
│  │  │ • 用于分流处理、异常数据输出                               │ │   │

## Transformation 与 Operator

### Transformation 概念

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Transformation 概念                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  Transformation 是 DataStream API 的底层表示：                           │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  用户代码：                                                     │   │
│  │  DataStream<String> stream = env.addSource(...)                │   │
│  │      .map(...)                                                 │   │
│  │      .filter(...);                                             │   │
│  │                                                                 │   │
│  │  内部转换：                                                     │   │
│  │  ┌───────────────────────────────────────────────────────────┐ │   │
│  │  │                                                           │ │   │
│  │  │  SourceTransformation                                     │ │   │
│  │  │        │                                                  │ │   │
│  │  │        ▼                                                  │ │   │
│  │  │  OneInputTransformation (Map)                             │ │   │
│  │  │        │                                                  │ │   │
│  │  │        ▼                                                  │ │   │
│  │  │  OneInputTransformation (Filter)                          │ │   │
│  │  │        │                                                  │ │   │
│  │  │        ▼                                                  │ │   │
│  │  │  SinkTransformation                                       │ │   │
│  │  │                                                           │ │   │
│  │  └───────────────────────────────────────────────────────────┘ │   │
│  │                                                                 │   │
│  │  Transformation 类型：                                          │   │
│  │  • SourceTransformation: 数据源                                │   │
│  │  • OneInputTransformation: 单输入转换                          │   │
│  │  • TwoInputTransformation: 双输入转换                          │     │
│  │  • PartitionTransformation: 分区转换                          │   │
│  │  • SinkTransformation: 数据输出                               │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  Transformation vs Operator：                                           │
│  • Transformation: 逻辑表示，描述数据流拓扑                             │
│  • Operator: 物理执行，实际处理数据的算子                               │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 常用 Operator

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        常用 Operator 分类                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 数据转换 Operator                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │

│  │  • map: 一对一转换                                              │   │

│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  2. 聚合 Operator                                                       │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • keyBy: 分区                                                  │   │
│  │  • reduce: 归约                                                 │   │
│  │  • sum: 求和                                                    │   │
│  │  • min/max: 最小/最大值                                         │   │
│  │  • aggregate: 自定义聚合                                        │   │
│  │                                                                 │   │

│  │  • join/cogroup: 流关联                                        │   │

│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  4. 时间相关 Operator                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • window: 窗口                                                 │   │
│  │  • windowAll: 非分区窗口                                        │   │
│  │  • assignTimestampsAndWatermarks: 时间戳和水位线                │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │

## 数据交换模式

### 数据交换类型

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        数据交换模式                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. Forward（前向传递）                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • 一对一传递                                                   │   │
│  │  • 不涉及网络传输（本地传递）                                   │   │
│  │  • 通常在 Operator Chain 内使用                                 │   │
│  │                                                                 │   │
│  │  ┌─────┐         ┌─────┐                                       │   │
│  │  │ A:1 │────────►│ B:1 │                                       │   │
│  │  │ A:2 │────────►│ B:2 │                                       │   │
│  │  └─────┘         └─────┘                                       │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  2. Shuffle（洗牌）                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • 随机分配到下游                                               │   │
│  │  • 涉及网络传输                                                 │   │
│  │  • 用于负载均衡                                                 │   │
│  │                                                                 │   │
│  │  ┌─────┐    ┌─────┐                                            │   │
│  │  │ A:1 │──┬─►│ B:1 │                                            │   │
│  │  │ A:2 │  │  │ B:2 │                                            │   │
│  │  └─────┘  └──┴─────┘                                            │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  3. Hash Partition（哈希分区）                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • 按 Key 哈希分配                                              │   │
│  │  • 相同 Key 发送到同一下游                                      │   │
│  │  • KeyBy 使用此模式                                             │   │
│  │                                                                 │   │
│  │  ┌─────┐    ┌─────┐                                            │   │
│  │  │ A:1 │──┬─►│ B:1 │ (Key=A)                                   │   │
│  │  │ B:2 │  │  │ B:2 │ (Key=B)                                   │   │
│  │  └─────┘  └──┴─────┘                                            │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  4. Broadcast（广播）                                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • 数据发送到所有下游                                           │   │
│  │  • 用于维度表广播                                               │   │
│  │                                                                 │   │
│  │  ┌─────┐    ┌─────┐                                            │   │
│  │  │ A   │──┬─►│ B:1 │                                            │   │
│  │  └─────┘  └─►│ B:2 │                                            │   │
│  │              └─────┘                                            │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  5. Rebalance（重平衡）                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • 轮询分配到下游                                               │   │
│  │  • 用于负载均衡                                                 │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  6. Rescale（重新缩放）                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • 仅与部分下游通信                                             │   │
│  │  • 减少网络连接数                                               │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### KeyBy 详解

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        KeyBy 分区原理                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  KeyBy 是最常用的分区操作：                                              │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  分区公式：                                                     │   │
│  │  partition = hash(key) % parallelism                            │   │
│  │                                                                 │   │
│  │  示例（parallelism = 4）：                                      │   │
│  │                                                                 │   │
│  │  数据：                                                         │   │
│  │  {key: "user_001", value: 100}                                 │   │
│  │  {key: "user_002", value: 200}                                 │   │
│  │  {key: "user_003", value: 300}                                 │   │
│  │  {key: "user_001", value: 150}                                 │   │
│  │                                                                 │   │
│  │  分区结果：                                                     │   │
│  │  ┌─────────────────────────────────────────────────────────┐  │   │
│  │  │ Partition 0          Partition 1                        │  │   │
│  │  │ user_001: [100,150]  user_002: [200]                    │  │   │
│  │  ├─────────────────────────────────────────────────────────┤  │   │
│  │  │ Partition 2          Partition 3                        │  │   │
│  │  │ user_003: [300]      (空)                                │  │   │
│  │  └─────────────────────────────────────────────────────────┘  │   │
│  │                                                                 │   │
│  │  特点：                                                         │   │
│  │  • 相同 Key 的数据一定在同一分区                               │   │
│  │  • 支持 Keyed State                                            │   │
│  │  • 支持 Watermark 对齐                                         │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  注意：KeyBy 不是物理上的分组，而是逻辑上的分区                          │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 总结

本章深入解析了 Flink 数据流模型：

| 概念 | 说明 |
|------|------|
| DataStream | 数据流抽象，支持各种转换操作 |
| Transformation | 逻辑表示，描述数据流拓扑 |
| Operator | 物理执行，实际处理数据的算子 |
| 数据交换 | Forward/Shuffle/Hash/Broadcast/Rebalance |
