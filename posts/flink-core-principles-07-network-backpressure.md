## 反压机制

### 反压原理

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        反压原理                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  反压（Backpressure）是下游处理慢导致上游降速的机制：                     │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  正常情况：                                                     │   │
│  │                                                                 │   │
│  │  Source ──────► Map ──────► Filter ──────► Sink                │   │
│  │    │             │            │             │                   │   │
│  │    ▼             ▼            ▼             ▼                   │   │
│  │  [数据流动顺畅]                                                 │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  反压情况：                                                     │   │
│  │                                                                 │   │
│  │  Source ──────► Map ──────► Filter ──────► Sink (慢)           │   │
│  │    │             │            │             │                   │   │
│  │    │             │            │      Buffer Full                │   │
│  │    │             │      Buffer Full            │                │   │
│  │    │       Buffer Full                          │                │   │
│  │    │                                             │                │   │
│  │    ▼                                             ▼                │   │
│  │  [反压向上游传播，整体降速]                                     │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  反压的好处：                                                           │
│  • 防止下游被压垮                                                       │
│  • 保证系统稳定性                                                       │
│  • 自动平衡处理速度                                                     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### Credit-based 流控

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Credit-based 流控                                 │|
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  Flink 1.5+ 使用 Credit-based 流控替代旧的反压机制：                      │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  原理：下游告知上游可发送的数据量（Credit）                       │   │
│  │                                                                 │   │
│  │  Sender (上游)                     Receiver (下游)              │   │
│  │  ┌─────────────┐                 ┌─────────────┐               │   │
│  │  │             │                 │             │               │   │
│  │  │   Buffer    │                 │   Buffer    │               │   │
│  │  │   Pool      │                 │   Queue     │               │   │
│  │  │             │                 │             │               │   │
│  │  └──────┬──────┘                 └──────┬──────┘               │   │
│  │         │                               │                       │   │
│  │         │  ──── Data (with backlog) ──► │                       │   │
│  │         │                               │                       │   │
│  │         │  ◄─── Credit Announcement ──── │                       │   │
│  │         │                               │                       │   │
│  │         │  ──── Data (≤ Credit) ──────► │                       │   │
│  │         │                               │                       │   │
│  │                                                                 │   │
│  │  交互流程：                                                     │   │
│  │  1. Receiver 告知 Sender 自己有多少可用 Buffer (Credit)         │   │
│  │  2. Sender 发送数据时携带 backlog (待发送数据量)                 │   │
│  │  3. Sender 只发送 Credit 允许的数据量                           │   │
│  │  4. Receiver 处理数据后，发送新的 Credit                        │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  优势：                                                                 │
│  • 更精确的流控，避免过度发送                                            │
│  • 减少网络拥塞                                                         │
│  • 更快的反压响应                                                       │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 反压诊断

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        反压诊断                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  诊断工具：                                                             │
│                                                                         │
│  1. Flink Web UI                                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  Backpressure 状态：                                            │   │
│  │  ┌───────────────────────────────────────────────────────────┐ │   │
│  │  │ OK (0-10%)      : 绿色，无反压                             │ │   │
│  │  │ Low (10-40%)    : 黄色，轻微反压                           │ │   │
│  │  │ High (40-100%)  : 红色，严重反压                           │ │   │
│  │  └───────────────────────────────────────────────────────────┘ │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │

│  2. 常见原因                                                           │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  • Sink 外部系统慢（数据库、Kafka）                             │   │
│  │  • 计算密集型操作（复杂逻辑、正则）                             │   │
│  │  • 窗口聚合、Join 操作                                          │   │
│  │  • GC 频繁                                                      │   │
│  │  • 数据倾斜                                                     │   │
│  │                                                                 │   │
```
